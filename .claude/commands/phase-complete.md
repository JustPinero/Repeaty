---
description: Mark current phase complete and run all phase-end audits. Usage `/phase-complete [phase-number]`.
---

Sequence:

1. **Verify all requests in the phase are done.** Read `requests/phase-N-*/` and check that every request file has a matching commit on the phase branch. List any that are missing or in-flight.

2. **Run `/run-audits N quick`** (delegates to audit-runner with all four skills).

3. **Update `e2e-manifest.json`** — flip every flow whose phase matches `N` from `not-started`/`in-progress` to `complete` if its E2E spec exists and passes locally.

4. **Decide:**
   - **No blocking findings** → recommend merging the phase branch to `main` and creating the next phase branch (`phase-{N+1}-{name}`). Present the user with the commit summary and ask before pushing.
   - **Blocking findings present** → STOP. Present the fix requests generated by the audits, recommend addressing them on the current branch before phase merge.

5. Once merged: prime the next phase by reading the next phase's overview (from the kickoff brief in `references/architecture.md` § Phases) and propose the first request to tackle.

Never auto-merge. The main branch's stability is the responsibility of the human in the loop.
