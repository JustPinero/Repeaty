# Requests

The live build plan. Each phase has a folder; each request inside a phase becomes one commit on the phase branch.

- `phase-1-foundation/` — full request set authored at kickoff (1.1–1.5).
- `phase-2-flashcards/` through `phase-6-pwa-launch/` — folder READMEs sketch the plan; full request files are authored at the start of each phase, after the prior phase's `/phase-complete` lands a clean merge.
- `phase-N-fixes/` — auto-generated by the audit skills (`/test-audit`, `/bughunt`, `/optimize`, `/drift-audit`). Filename pattern: `fix-<type>-<short-desc>.md`.

Every request file follows the same structure:
- **Objective**
- **Acceptance Criteria → Test Mapping** (table)
- **RED Phase** (failing tests to write first)
- **GREEN Phase** (implementation steps)
- **Files to Touch** (new + updated)
- **Dependencies**
- **Out of scope**
