# Request 1.3 — Supabase Auth + Signup/Login UI

## Objective
Wire Supabase Auth (email + password) into the React app. Build accessible signup and login pages, an auth provider that exposes the current user via TanStack Query, route guards for authenticated-only routes, and a "verify your email" interstitial. After this request, a user can sign up, confirm email (via the Supabase Inbucket inbox locally), log in, and reach a placeholder `/app` route.

## Acceptance Criteria → Test Mapping

| Criterion                                                              | Test                                                                       |
| ---------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Signup form validates email + password (min 8 chars) before submission | `SignupForm.test.tsx` — Zod schema rejects bad input; form does not submit |
| Submitting valid signup calls `supabase.auth.signUp` once              | `SignupForm.test.tsx` with mocked Supabase client                          |
| Login form rejects bad creds with a visible error message              | `LoginForm.test.tsx` — error region appears with `role="alert"`            |
| `useAuthUser()` hook returns the current user (or null)                | `useAuthUser.test.ts` against a faked Supabase auth listener               |
| `<RequireAuth>` redirects unauthenticated users to `/login`            | `RequireAuth.test.tsx` — assert redirect via `useNavigate` mock            |
| All form inputs have associated `<label htmlFor>`                      | `SignupForm.test.tsx` and `LoginForm.test.tsx` use `getByLabelText`        |
| Error region is keyboard- and screen-reader-reachable                  | a11y assertion: `role="alert"`, focused on appearance                      |
| E2E: signup → email confirm (Inbucket) → login → reach `/app`          | `apps/web/tests/e2e/signup-and-onboarding.spec.ts` (partial — finishes in 1.4) |
