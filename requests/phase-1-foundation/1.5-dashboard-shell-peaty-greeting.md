# Request 1.5 — Dashboard Shell + Peaty Greeting

## Objective
Build the authenticated `/app` dashboard shell: header with user name, "Hi, I'm Peaty" greeting card with the mascot illustration from `assets/peaty/`, a placeholder review queue ("0 cards due — your decks land in Phase 2"), language selector (when user has > 1 target language), logout. After this request, signup → onboarding → dashboard is a complete loop and Phase 1's exit criteria are met.

## Acceptance Criteria → Test Mapping

| Criterion                                                                          | Test                                                                                                              |
| ---------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| Dashboard greets the user by `display_name`                                        | `Dashboard.test.tsx` — `getByRole('heading', { name: /Hi, <name>/ })`                                             |
| Peaty illustration renders with meaningful alt text                                 | `PeatyGreeting.test.tsx` — `getByAltText('Peaty the parrot waving hello')`                                        |
| Language selector appears only when user has > 1 target language                   | `LanguageSelector.test.tsx` — single language: not rendered; multiple: select with all of them                     |
| Selecting a language persists the choice in Zustand and survives page refresh      | `useActiveLanguage.test.ts` — Zustand persist middleware to localStorage                                          |
| Placeholder review queue shows "0 cards due" with a "Bundled decks land in Phase 2" hint | `ReviewQueuePlaceholder.test.tsx`                                                                          |
| Logout button signs out via Supabase and redirects to `/login`                     | `Header.test.tsx` — mock `supabase.auth.signOut`; assert call + redirect                                          |
| Header user-menu is keyboard-reachable; menu items have `role="menuitem"`          | a11y test                                                                                                          |
| Lighthouse accessibility score ≥ 90 on `/app` (manual check, recorded in PR)       | Playwright axe scan in E2E asserts no Critical violations                                                           |
| `e2e-manifest.json` marks `signup-and-onboarding` as `complete`                    | Manifest diff in PR + `/phase-complete` step                                                                       |
