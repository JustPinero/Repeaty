# Request 2.4 ā€” Review Session Flow + Rating Buttons

## Objective
Build the heart of Phase 2: a review session that pulls the user's due cards, presents each with the existing `<Flashcard>`, captures Again/Hard/Good/Easy ratings, runs them through the FSRS scheduler from `@repeaty/shared`, and persists the updated `reviews` row to Supabase. After this request, a user can complete a full review session on the bundled Spanish or French A1 deck.

## GREEN Phase

1. **`useReviewSession(deckId)`** hook in `apps/web/src/features/review/`:
   - Fetches the deck's cards via supabase, joined with the user's `reviews` rows (left join on `user_id = auth.uid() AND card_id = ...`).
   - Cards without a review row use `initialState(now)` from `@repeaty/shared`.
   - Builds an in-memory queue: due cards first (sorted by `due_at ASC`), then new cards.
   - Exposes `currentCard`, `progress`, `submitRating(rating)`, `isComplete`.
   - `submitRating(rating)`:
     - Calls `schedule(currentState, rating, now)` from shared.
     - Upserts the `reviews` row with the new state (deno-isomorphic supabase-js call).
     - Updates the queue: Again ā†’ re-enqueue at back; others ā†’ advance.
     - Marks `isComplete` when the queue is empty.
   - Tracks per-session stats: `reviewedCount`, `correctCount` (Hard/Good/Easy = correct, Again = incorrect for this aggregation).

2. **`<RatingButtons>`** component:
   - Four buttons: Again (red), Hard (amber), Good (green), Easy (blue).
   - Each has aria-label + data-testid.
   - Global keyboard listener inside the page maps 1ā€“4 to the four ratings (only when no input is focused).

3. **`<ReviewSessionPage>`** at route `/app/decks/:deckId/review`:
   - Mounts inside `<RequireAuth><OnboardingGuard>` like other /app routes.
   - Renders `<Flashcard>` + `<RatingButtons>` while `!isComplete`.
   - Renders a session-complete summary (cards reviewed, accuracy, "Back to dashboard" link) when complete.
   - Empty-queue state: "Nothing due ā€” come back later" + link.

4. **DeckListItem update**: the link from a deck list item now goes to `/app/decks/:id/review` (or to a deck-detail page that shows the review CTA ā€” pick one; review-direct is simpler for v1).

5. **Dashboard's review queue**: replace the placeholder with a real query that counts due cards and links to the highest-priority deck's review session. (The placeholder text is now misleading ā€” Phase 2 IS shipping decks, so update it.)
