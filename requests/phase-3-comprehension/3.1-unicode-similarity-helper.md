# Request 3.1 — Unicode-Aware Similarity Helper in `@repeaty/shared`

## Objective
Build a small, well-tested string-similarity helper in `@repeaty/shared/similarity.ts` that the comprehension session (Request 3.2) and the pronunciation Whisper transcript scorer (Phase 4) will both rely on. The function compares a learner's response against the expected target text and returns a 0.0–1.0 similarity score. Critically: it must handle Unicode normalization (NFC), case-fold, ignore diacritics for ES/FR/IT/DE/PT (so "café" ≈ "cafe" still scores high), and treat Cyrillic correctly (Russian Ё/Е, ѝ/й are real semantic distinctions — don't fold them).

## Acceptance Criteria → Test Mapping

| Criterion                                                                       | Test                                                                       |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Identical strings score 1.0                                                     | `similarity.test.ts > exact match`                                         |
| Empty target throws (programming error)                                         | `similarity.test.ts > empty target`                                        |
| Empty response scores 0.0                                                       | `similarity.test.ts > empty response`                                      |
| Trims leading/trailing whitespace before comparing                              | `similarity.test.ts > whitespace`                                          |
| Case-insensitive: `"Hola" === "hola"` → 1.0                                    | `similarity.test.ts > case`                                                |
| NFC-normalized: `"café"` (composed) and `"café"` (decomposed) both match | `similarity.test.ts > nfc round-trip`                                      |
| ES/FR/IT/DE/PT: diacritic-folded — `"adios"` ≈ `"adiós"` scores 1.0            | `similarity.test.ts > latin diacritics fold`                               |
| Russian (`ru`): Ё ≠ Е, Й ≠ И — no fold                                         | `similarity.test.ts > russian no-fold`                                     |
| Japanese / Mandarin: full-width ↔ half-width handled (ーvs-)                    | `similarity.test.ts > cjk normalization`                                   |
| Off-by-one typo on a 5-char word: ≥ 0.6 (Levenshtein-derived)                  | `similarity.test.ts > one-typo`                                            |
| Wholly different strings: 0.0                                                   | `similarity.test.ts > unrelated`                                           |
| Function is pure: same input → same output                                      | `similarity.test.ts > pure`                                                |

## RED Phase

`packages/shared/src/similarity.test.ts` covers all 12 assertions against a stub `similarity()` that returns 0.

## GREEN Phase

1. **Implement** `packages/shared/src/similarity.ts`:
   ```ts
   export type SimilarityOptions = {
     /** BCP-47 code. Drives diacritic-fold rules. Defaults to safe (no fold). */
     lang?: string;
   };

   export function similarity(target: string, response: string, opts?: SimilarityOptions): number;
   ```
2. **Algorithm** (v1, intentionally simple):
   - Validate target is non-empty (throw on empty target).
   - Trim both inputs.
   - If response is empty, return 0.
   - NFC-normalize both.
   - Casefold (`.toLocaleLowerCase()` if `lang` given, else `.toLowerCase()`).
   - **Diacritic fold** for `lang ∈ { es, fr, it, de, pt }`: NFD-decompose, strip `\p{M}` (combining marks), NFC-recompose. Skip for `ru / ja / zh`.
   - Fold full-width to half-width for `ja` / `zh` via NFKC.
   - Compute normalized Levenshtein:
     - `1 - levenshtein(a, b) / max(a.length, b.length)`.
   - Clamp to `[0, 1]`.
3. **Re-export** from `packages/shared/src/index.ts`.
