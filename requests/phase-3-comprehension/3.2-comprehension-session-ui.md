# Request 3.2 — Comprehension Session UI

## Objective
Build the user-facing comprehension session at `/app/decks/:deckId/comprehension`: a flow where the user sees a target-language word/phrase, types its translation, and gets a per-card score combining accuracy and response time. Reuses the deck infrastructure from Phase 2.

## Acceptance Criteria → Test Mapping

| Criterion                                                                          | Test                                                                       |
| ---------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| `/app/decks/:deckId/comprehension` route is auth-gated + onboarding-gated          | route config + existing wrappers                                           |
| First card shows `target_text` + a text input + a Submit button                    | `ComprehensionSessionPage.test.tsx`                                        |
| Submit fires onSubmit with the (trimmed) response and elapsed-ms                   | `useComprehensionSession.test.ts`                                          |
| Enter submits when the input is focused and non-empty                              | component test                                                              |
| After submit: per-card result is shown (correct/close/miss + similarity score)     | `ComprehensionSessionPage.test.tsx`                                        |
| Soft timer: a non-blocking ticking display ("3.4s" or similar) — no hard cutoff    | `Timer.test.tsx`                                                            |
| Empty queue → "Nothing due — try again later." (consistent with review session)    | page test                                                                   |
| Completion summary shows accuracy% + average response time                         | page test                                                                   |
| Cyrillic input correctly handled in the similarity comparison                      | integration via `similarity` (3.1) — covered by similarity.test.ts          |

## GREEN Phase

1. `useComprehensionSession(deckId)` — analogous to `useReviewSession` but the rating is computed from the response, not chosen by the user. Reuses the same nested-select pattern (cards + per-user comprehension_attempts joined). Records a per-card `response_ms`.
2. `<Timer>` — small component using `requestAnimationFrame` to update the displayed elapsed time without re-rendering the rest of the page.
3. `<ComprehensionSessionPage>` — input, Submit, per-card result, completion summary.
4. Wire into routes: `/app/decks/:deckId/comprehension`.
5. DeckListItem grows a second link (or a single link to a deck-detail page that picks the mode). Decision: stick with the `/review` link as primary; add a secondary "Comprehension" sublink in DeckListItem.
