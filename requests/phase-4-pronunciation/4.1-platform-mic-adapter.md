# Request 4.1 — Platform Mic Adapter (Web)

## Objective
Extend `apps/web/src/platform/` with mic-capture methods (`requestMicPermission`, `startRecording`, `stopRecording`, `playRecordedAudio`). The `PlatformAdapter` interface already has these commented out — uncomment + implement against `MediaRecorder`. Capacitor swap (DEBT-002) gets the same surface; the abstraction stays clean.

## Acceptance Criteria → Test Mapping

| Criterion                                                                       | Test                                                                       |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| `canRecord()` returns true iff `navigator.mediaDevices.getUserMedia` + `MediaRecorder` exist | `web.test.ts > canRecord true`                                |
| `canRecord()` returns false in jsdom / SSR / hostile envs                       | `web.test.ts > canRecord false`                                            |
| `requestMicPermission()` returns `'granted' \| 'denied' \| 'prompt'` per `permissions.query` | `web.test.ts > permission states`                              |
| `startRecording()` returns a `RecordingHandle` with `MediaRecorder` started     | `web.test.ts > startRecording`                                             |
| `stopRecording(handle)` resolves a Blob (audio/webm or audio/mp4 on iOS Safari) | `web.test.ts > stopRecording produces Blob`                                |
| `playRecordedAudio(blob)` plays via `<audio>` and resolves on `ended`           | `web.test.ts > playRecordedAudio`                                          |
| All methods reject if mic permission was denied                                 | `web.test.ts > denied`                                                     |
| `cancelRecording(handle)` aborts mid-record (no Blob returned)                  | `web.test.ts > cancel`                                                     |

## RED Phase

`apps/web/src/platform/web.test.ts` extended with the new method coverage. Stub web.ts implementations throw / return wrong types until GREEN.

## GREEN Phase

1. Update `PlatformAdapter` (in `types.ts`) — uncomment the 4 mic methods + add `canRecord` + `cancelRecording`.
2. Implement in `web.ts`:
   - `canRecord()` checks `navigator.mediaDevices?.getUserMedia` + `window.MediaRecorder`.
   - `requestMicPermission()` queries `navigator.permissions.query({ name: 'microphone' as PermissionName })`. Falls through to `'prompt'` if Permissions API absent.
   - `startRecording()` calls `getUserMedia({ audio: true })`, instantiates `MediaRecorder` with the best available mime type (`audio/webm;codecs=opus` preferred; `audio/mp4` fallback for iOS Safari), starts recording. Returns `{ recorder, stream, chunks: Blob[] }`.
   - `stopRecording(handle)` awaits the `dataavailable` + `stop` events, stops the stream, returns the assembled Blob.
   - `cancelRecording(handle)` stops the stream without resolving a Blob.
   - `playRecordedAudio(blob)` creates an `Audio` element from a `URL.createObjectURL`, plays it, resolves on `ended`.
