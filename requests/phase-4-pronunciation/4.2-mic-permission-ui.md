# Request 4.2 — Mic Permission UX + Recording Controls

## Objective
A reusable `<MicCapture>` component that handles the full mic UX: permission prompt, recording state, stop button, playback of the recorded blob, re-record. Per the deployment-landmines doc, iOS Safari requires the permission request to come from a user gesture — the component's start-record button satisfies that.

## Acceptance Criteria → Test Mapping

| Criterion                                                                       | Test                                                                       |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Initial state: shows a "🎤 Record" button                                        | `MicCapture.test.tsx > initial`                                            |
| Click → `platform.requestMicPermission()` then `platform.startRecording()`     | `MicCapture.test.tsx > start flow`                                         |
| Mid-record: shows a "⏹️ Stop" button + an elapsed-time display (Timer reuse)   | `MicCapture.test.tsx > recording`                                          |
| Click Stop → resolves the recorded Blob via `onRecorded` callback               | `MicCapture.test.tsx > onRecorded`                                         |
| Permission denied → shows an inline message + a "Try again" button              | `MicCapture.test.tsx > denied`                                             |
| `canRecord()` false → renders a fallback "Recording isn't supported" message    | `MicCapture.test.tsx > unsupported`                                        |
| Playback button appears after recording; click plays via `platform.playRecordedAudio` | `MicCapture.test.tsx > playback`                                     |

## RED Phase

`apps/web/src/features/pronunciation/MicCapture.test.tsx` against a stub `<MicCapture>` that renders a placeholder.

## GREEN Phase

1. State machine:
   - `idle` (default), `requesting-permission`, `recording`, `stopping`, `recorded`, `denied`, `unsupported`, `error`.
2. Reuse `Timer` from `apps/web/src/features/comprehension/Timer.tsx` for the elapsed display.
3. Component is **fully props-driven** (no hooks for state outside its own React state) so it's reusable from anywhere — the pronunciation session, future card-detail review widgets, etc.
4. Mock-friendly: `@/platform` is the only external dependency; tests mock its methods.
