# Request 4.3 — Audio Storage + Path-Prefix RLS

## Objective
Provision a `pronunciation-audio` Supabase Storage bucket with path-prefix RLS so user A can never read or write under user B's prefix. Add a thin `uploadPronunciationBlob(blob, cardId, userId)` helper in `apps/web/src/features/pronunciation/storage.ts` that uploads to `${userId}/${cardId}/${uuidv4}.{webm|mp4}` and returns the storage path.

## GREEN Phase

1. **Migration** `supabase/migrations/0011_pronunciation_audio_bucket.sql`:
   - `insert into storage.buckets (id, name, public) values ('pronunciation-audio', 'pronunciation-audio', false) on conflict do nothing;`
   - Path-prefix RLS via `(SELECT auth.uid())::text = (storage.foldername(name))[1]` for SELECT/INSERT/UPDATE/DELETE on `storage.objects` where bucket_id = 'pronunciation-audio'.
2. **Helper** `apps/web/src/features/pronunciation/storage.ts`:
   - `uploadPronunciationBlob(blob, { userId, cardId })` → calls `supabase.storage.from('pronunciation-audio').upload(path, blob)`, returns the path.
   - 10 MB size check before upload.
   - Mime-type sniffed from the Blob (`audio/webm`, `audio/mp4`, etc.).
