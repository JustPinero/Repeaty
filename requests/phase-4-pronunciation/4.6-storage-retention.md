# Request 4.6 — Storage Retention (free tier: 7-day audio cleanup)

## Objective
Per `references/schema.md` and the privacy section of the original brief: free-tier audio recordings auto-expire after 7 days. The `pronunciation_attempts.audio_storage_path` column is set to NULL when the file is reaped, but the row stays for history. Pro-tier users keep audio indefinitely.

## Acceptance Criteria → Test Mapping

| Criterion                                                                       | Test                                                                       |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Files older than 7 days under `pronunciation-audio/${user_id}/...` are deleted IF the user is `tier = 'free'` | integration: `audio-retention.test.ts` (manipulates created_at and runs the job) |
| Pro/admin users' audio is preserved past 7 days                                 | same                                                                       |
| The matching `pronunciation_attempts` row's `audio_storage_path` is NULLed       | same                                                                       |
| Job is idempotent — running twice produces the same outcome                     | same                                                                       |
| Job has structured logging matching the api-contracts logging contract          | manual                                                                     |

## RED Phase

- `apps/web/tests/integration/supabase/audio-retention.test.ts`

## GREEN Phase

Two paths — pick one based on what's available on Supabase Cloud:

**Path A — pg_cron + a SQL function:**
- Migration `0012_audio_retention.sql`:
  - SQL function `purge_free_tier_audio()` that joins `pronunciation_attempts` × `profiles` × `storage.objects` for tier='free' and `created_at < now() - interval '7 days'`. Calls `storage.delete_object` for each file. Sets `audio_storage_path = NULL` on the attempt row.
  - `cron.schedule('audio-retention-daily', '0 3 * * *', $$ select public.purge_free_tier_audio() $$)`.
- Tests: integration test inserts old rows + runs the function manually.

**Path B — scheduled Edge Function:**
- `supabase/functions/audio-retention/index.ts` doing the same logic in Deno.
- Scheduled via Supabase Cron from the dashboard (manual setup; document in `references/deployment-landmines.md`).

Recommend **Path A** — keeps the logic colocated with the schema migrations; no Edge-Function cold-start cost.
