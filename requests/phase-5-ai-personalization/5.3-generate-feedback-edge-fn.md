# Request 5.3 — `generate-feedback` Edge Function (Claude proxy)

## Objective
Second Edge Function. Pro/admin-only. Caller posts `{ kind, attempt_id }` for a non-perfect comprehension or pronunciation attempt; the function loads the attempt + card + caller's `native_language_code` + `cefr_level`, computes an `error_pattern` key, checks `feedback_cache` (5.1), and either returns the cached `feedback_text` or calls Claude with a structured JSON prompt and writes both `feedback_cache` and the source attempt's `feedback_text`.

## GREEN Phase

1. **Error-pattern keys:**
   - Pronunciation: hash of (Whisper transcript ↔ target diff edits, sorted, normalized via `@repeaty/shared/similarity`'s normalize).
   - Comprehension: bucket × normalized response distance.
2. **`supabase/functions/generate-feedback/handler.ts`:** factory deps for `getUserFromJwt`, `getProfile`, `getAttemptForUser`, `getFeedbackCache`, `insertFeedbackCache`, `updateAttemptFeedback`, `bumpRateLimit`, `callClaude`, `now`, `log`, `estimateClaudeCostUsd`. The `callClaude` dep wraps a 15s-AbortController POST to `/v1/messages` with a Zod-validated JSON output schema.
3. **`packages/shared/src/feedback-prompt.ts`:** the prompt template + Zod output schema. Importable from Deno via relative path.
4. **`supabase/functions/generate-feedback/index.ts`:** production wiring with the same `validateEnv` shape as `score-pronunciation` plus `ANTHROPIC_API_KEY`.
