# Request 5.4 — Phase-5 `useFeedback` Claude Swap

## Objective
Replace the Phase-3 canned-text body of `useFeedback` with a TanStack-Query call to the `generate-feedback` Edge Function (5.3). Per `references/api-contracts.md`'s explicit warning: **public hook types stay the same**. Callers (`FeedbackPanel`, `ComprehensionSessionPage`, `PronunciationSessionPage`) must not need to change.

## Acceptance Criteria → Test Mapping

| Criterion                                                                       | Test                                                                       |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Public types `FeedbackInput` + `FeedbackResult` unchanged                       | `useFeedback.test.ts` (existing tests still pass)                          |
| Free-tier caller: hook resolves with `text: null, isLoading: false` (no fetch)  | new test                                                                   |
| Pro-tier caller, perfect bucket: same — no fetch                                | new test                                                                   |
| Pro-tier caller, miss bucket: invokes `generate-feedback` once and surfaces the response text | new test                                                       |
| Edge Function rate-limit (429) error: hook returns `text: null, isLoading: false` and logs warning (no UI surface) | new test                                          |
| Edge Function timeout: same fallback                                            | same                                                                       |
| Re-render with the same input doesn't re-fetch (TanStack staleTime + queryKey) | same                                                                       |

## RED Phase
- New cases inside the existing `useFeedback.test.ts`. `FeedbackPanel.test.tsx` should also stay green without modification.

## GREEN Phase

1. Detect tier via `useAuthUser` → join with `profiles.tier`. (Profile is already loaded by Phase 1's auth context; if not, extend the context.)
2. Wrap the `useQuery` body around `supabase.functions.invoke('generate-feedback', { body: { kind, attempt_id } })`. Note: `attempt_id` isn't in `FeedbackInput` today — extend the input shape with `attemptId` (additive; existing callers pass it after 5.3 lands and the FeedbackPanel callsites get the id from the session result).
3. queryKey: `['feedback', kind, attempt_id]`. staleTime 5 min (cached feedback for the same attempt doesn't change).
4. On free-tier or perfect-bucket: short-circuit to `{ text: null, isLoading: false }` without hitting the network.
5. On 429 / timeout / error: log to console, return `{ text: null, isLoading: false }`. Don't surface a red error to the user — the textual answer is still readable; the AI feedback is a nice-to-have.
