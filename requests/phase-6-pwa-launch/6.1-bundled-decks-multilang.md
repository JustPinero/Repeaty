# Request 6.1 — Bundle the remaining 5 starter decks (de, it, ru, ja, zh)

## Objective
Phase 2 bundled `es` + `fr` only. The kickoff brief promises starter decks for all 7 languages (es, fr, de, it, ru, ja, zh) shipped with the app. This request authors the deck-spec YAML files for the remaining 5, regenerates `0009_seed_bundled_decks.sql` via `scripts/seed/seed-decks.ts`, and verifies bundled-deck visibility under RLS for each language.

## Acceptance Criteria → Test Mapping

| Criterion                                                                       | Test                                                                       |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Each of `de`, `it`, `ru`, `ja`, `zh` has a Starter deck (A1, ~30 cards)         | `bundled-decks.test.ts` extension                                          |
| Each card has target_text, native_text, and an optional ipa for ja/zh           | `seed-decks.test.ts` (existing schema test, extended for new langs)        |
| Cards survive `supabase db reset` (seed migration is byte-stable across runs)   | existing UUIDv5 determinism test                                           |
| Onboarding lets a user pick any of the 7 target languages and lands on the matching starter deck | E2E extension                                                |
| Per-language similarity normalization smoke-test on bundled cards                | `similarity.test.ts` (existing — confirm ja/zh NFKC fold, ru no diacritic fold) |

## RED Phase
- Extend `bundled-decks.test.ts` to assert per-language deck count > 0 and per-language card count >= 25.

## GREEN Phase

1. **Author 5 deck-spec YAMLs** at `scripts/seed/decks/{de,it,ru,ja,zh}-starter-a1.yaml`. Each file: 30 high-frequency A1 vocabulary cards with `target_text`, `native_text` (in English), optional `ipa` for ja/zh, optional `example_sentence_target` / `example_sentence_native` for verbs.
2. **Run** `pnpm seed:decks` to regenerate `0009_seed_bundled_decks.sql`. The file is auto-generated; the diff should be five new card-block sections.
3. **Verify locally** with `supabase db reset --no-seed && supabase db push` that the seed applies cleanly + the cards land.
4. Update `references/architecture.md` ADR section if any decision (CEFR level, deck size, source of word lists) needed re-confirming.
