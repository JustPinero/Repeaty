# Request 6.3 — Service Worker + Workbox Caching

## Objective
Register a Workbox-built service worker that precaches static assets (JS/CSS/fonts), uses StaleWhileRevalidate for bundled deck JSON, and NetworkOnly (with the offline queue from 6.4 wrapping retries) for Supabase API calls. Brings the PWA close to "works offline once visited once".

## Acceptance Criteria → Test Mapping

| Criterion                                                                       | Test                                                                       |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| `vite-plugin-pwa` (Workbox) wired into `vite.config.ts`                         | build smoke (Lighthouse 6.5)                                               |
| Service worker registers on app boot when in production                         | Lighthouse + manual cypress smoke                                          |
| Static assets (hashed filenames) precached on install                           | Lighthouse PWA audit                                                       |
| Bundled card images / JSON: StaleWhileRevalidate                                 | Workbox config inspected                                                   |
| Supabase API calls: NetworkOnly with offline-queue handler (handed off to 6.4)  | Workbox config inspected                                                   |
| Service worker scope is `/` (no over-broad scope)                               | Lighthouse                                                                 |

## RED Phase
- A Vitest smoke that imports the PWA plugin config and asserts the run-time config + manifest.

## GREEN Phase

1. **Add deps:** `vite-plugin-pwa` + workbox runtime (the plugin pulls in workbox-window).
2. **`vite.config.ts`** wires the plugin with:
   - `registerType: 'autoUpdate'`
   - `manifest` block (or `manifestFilename: 'manifest.webmanifest'` + use the static one from 6.2)
   - `workbox.runtimeCaching` rules:
     - `*.{js,css,woff2}` → CacheFirst, immutable
     - `/peaty/.*\\.(jpg|png)` → CacheFirst, 30-day expiration
     - Supabase REST `*.supabase.co/rest/v1/.*` → NetworkOnly
3. **Register the SW** in `apps/web/src/main.tsx` via `virtual:pwa-register`.
4. **Console-log dev-mode SW status** so the manual smoke is debuggable.
