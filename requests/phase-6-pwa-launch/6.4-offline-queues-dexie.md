# Request 6.4 — Offline Queues (Dexie)

## Objective
The user records a pronunciation, types a comprehension answer, or rates a flashcard while offline. The action is enqueued in IndexedDB (via Dexie) and replayed on reconnect. Three queues per `references/repeaty-pwa.md` § Offline queue: `pending_reviews`, `pending_pronunciation_attempts`, `pending_comprehension_attempts`.

## Acceptance Criteria → Test Mapping

| Criterion                                                                       | Test                                                                       |
| ------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| Three Dexie tables exist with the right schema                                  | unit test                                                                  |
| `enqueueReview(payload)`, `enqueueComprehension(...)`, `enqueuePronunciation(...)` helpers persist to IndexedDB | unit test                                                       |
| `replayQueues()` flushes all three in chronological order on reconnect          | unit test                                                                  |
| Replay handles Supabase 401 (re-auth) without losing items                       | unit test                                                                  |
| Replay handles per-row failures (RLS rejection) by leaving the bad item flagged + continuing | unit test                                                       |
| Conflict resolution: server wins; client overwrites only when client row strictly older | unit test                                                          |
| `useReviewSession.submitRating`, `useComprehensionSession.submitResponse`, `usePronunciationSession.submitRecording` fall through to the queue when offline | hook tests with simulated offline |
