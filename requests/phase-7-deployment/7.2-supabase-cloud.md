# Request 7.2 — Supabase Cloud Backend Deploy

## Objective
Get the Postgres schema + Edge Functions live on Supabase Cloud so the Vercel frontend has something to talk to. Six Edge Functions to deploy (`flip-tier`, `score-pronunciation`, `generate-feedback`, `generate-lesson`, `tts-jazh`, `audio-retention`); 20 migrations to apply; three secrets to set; one cron schedule to wire (`audio-retention` daily 03:30 UTC, can't be configured via the migration tree because it's an Edge Function not pg_cron).

## GREEN Phase

1. **One-time project creation (dashboard):**
   - Create a Supabase Cloud project named `repeaty`. Pick a region close to Ben (`us-east-1` or `us-west-2`).
   - Note the project ref (`<ref>` — visible in the URL: `app.supabase.com/project/<ref>`).
   - Enable `pg_cron` from Database → Extensions.

2. **Local CLI link + push:**
   ```bash
   supabase link --project-ref <ref>
   supabase db push                    # applies 0001..0020
   ```

3. **Deploy script** at `scripts/deploy-supabase.sh`:
   - Loops over `supabase/functions/*/index.ts` and runs `supabase functions deploy <name>` for each.
   - Skips `_shared/` (not a function — it's the helper module).
   - Verifies `supabase secrets list` includes the three required keys; warns + lists missing ones.
   - Idempotent — safe to re-run after a code change.

4. **Set secrets:**
   ```bash
   supabase secrets set \
     OPENAI_API_KEY=sk-... \
     ANTHROPIC_API_KEY=sk-ant-... \
     SUPABASE_SERVICE_ROLE_KEY=$(supabase status -o env --linked | awk -F= '/^SERVICE_ROLE_KEY=/{print $2}')
   ```
   (The service-role key on cloud is different from local; pull it from `--linked` flag's status output OR copy from the dashboard's API Settings.)

5. **Audio-retention cron** (manual):
   - Dashboard → Database → Cron → Create new job.
   - Name: `audio-retention-blob-cleanup`.
   - Schedule: `30 3 * * *` (daily 03:30 UTC).
   - SQL: `select net.http_post(url := '<edge-fn-url>/audio-retention', headers := jsonb_build_object('apikey', '<service-role-key>'));`
   - Note: requires `pg_net` extension. Enable from Database → Extensions if not already.
