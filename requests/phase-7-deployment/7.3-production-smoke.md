## Objective
Verify the deployed v1 actually works end-to-end against the production Vercel URL + Supabase Cloud project. One automated script to run after every deploy + a manual checklist of what to poke that the script can't cover (browser flows, mic, iOS install).

## GREEN Phase

1. **`scripts/post-deploy-smoke.sh <url>` <supabase-fn-url>**:
   ```bash
   #!/usr/bin/env bash
   set -euo pipefail
   URL="${1:?usage: post-deploy-smoke.sh <vercel-url> <supabase-functions-url>}"
   FN_URL="${2:?}"

   # Frontend smoke
   for path in "" "/login" "/signup" "/app"; do
     code=$(curl -sf -o /dev/null -w "%{http_code}" "$URL$path") || true
     [ "$code" = "200" ] || { echo "❌ $URL$path -> $code"; exit 1; }
   done
   curl -sf "$URL/manifest.webmanifest" | jq -e '.name == "Repeaty"' >/dev/null || \
     { echo "❌ manifest invalid"; exit 1; }
   curl -sfI "$URL/peaty/peaty-icon-192.png" >/dev/null || { echo "❌ icon-192 missing"; exit 1; }

   # Edge Functions: every one should 401 a no-auth POST.
   for fn in flip-tier score-pronunciation generate-feedback generate-lesson tts-jazh; do
     code=$(curl -sf -o /dev/null -w "%{http_code}" -X POST "$FN_URL/$fn") || true
     [ "$code" = "401" ] || { echo "❌ $fn unauth POST -> $code (expected 401)"; exit 1; }
   done
   # audio-retention requires the service-role apikey header — POST without it still returns 401.
   code=$(curl -sf -o /dev/null -w "%{http_code}" -X POST "$FN_URL/audio-retention") || true
   [ "$code" = "401" ] || { echo "❌ audio-retention unauth POST -> $code (expected 401)"; exit 1; }

   echo "✓ all smoke checks passed"
   ```
2. **Manual checklist** (append to `.claude/manual-testing.md` § Production):
   - Sign up a brand-new account at `<url>/signup`. Confirm email via the cloud project's Auth → Users page (or the configured SMTP if added).
   - Run through the free-tier critical-path smoke from manual-testing.md against the production URL.
   - Flip the new account to `tier: pro` via Studio (or via your admin account at `<url>/app/admin` if you've already promoted yourself).
   - Run a `/app/generate` request — confirm Anthropic key works + a deck lands.
   - Record a pronunciation attempt — confirm OpenAI Whisper key works + a similarity score returns.
   - JA/ZH TTS — open Studio → SQL Editor → `update profiles set tier='pro' where email='<you>'`, then play a Japanese card. DevTools Network tab should show a POST to `/functions/v1/tts-jazh`.
   - PWA install on Chrome desktop + Add-to-Home-Screen on iOS Safari.
3. **Rollback steps in `references/deployment-landmines.md`:**
   - **Vercel:** `vercel rollback` (instant promote of the prior production deployment).
   - **Supabase migrations:** forward-only — you can't `supabase db pull` your way out of a bad migration. The recovery path is a fresh follow-up migration that reverses the bad change. Plan accordingly.
   - **Supabase Edge Functions:** `supabase functions deploy <name>` from the prior commit (i.e. `git checkout <prev-sha> -- supabase/functions/<name> && supabase functions deploy <name> && git checkout main -- supabase/functions/<name>`).
