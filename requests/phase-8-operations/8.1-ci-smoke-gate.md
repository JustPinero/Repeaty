# Request 8.1 — CI Production Smoke Gate

## Objective
Add a GitHub Actions job that runs `pnpm smoke` against the live production deployment after every push to `main`. The job is the first line of defense against a `vercel.json` regression, a missing migration, or a removed Edge Function — anything `scripts/post-deploy-smoke.sh` already asserts. Today the script exists (DEBT-009 resolved 2026-05-04) but is only run by hand.

This is the prerequisite for 8.2 (auto-deploy): we won't auto-promote a deploy without an automated check that it answers correctly.

## RED Phase

A regression test for the workflow file itself: a tiny `scripts/test-ci-config.ts` (Vitest) that parses `.github/workflows/ci.yml` and asserts:
- A job with id `production-smoke` exists.
- Its `if:` is `github.ref == 'refs/heads/main' && github.event_name == 'push'`.
- It has no `needs:` field.
- The smoke step's `run:` command is exactly `pnpm smoke`.

The test fails RED today; passes once 8.1 ships.

## GREEN Phase

1. **Workflow job (`.github/workflows/ci.yml`):**
   ```yaml
   production-smoke:
     name: production smoke (post-push to main)
     runs-on: ubuntu-latest
     timeout-minutes: 5
     if: github.ref == 'refs/heads/main' && github.event_name == 'push'
     steps:
       - uses: actions/checkout@v6
       - uses: pnpm/action-setup@v6
       - uses: actions/setup-node@v6
         with:
           node-version: ${{ env.NODE_VERSION }}
           cache: pnpm
       - name: Install (smoke needs jq via apt — Ubuntu runner has it preinstalled)
         run: pnpm install --frozen-lockfile
       - name: Run production smoke
         run: pnpm smoke
   ```
   Notes:
   - No `needs:` — runs in parallel with the existing `validate` and `edge-functions` jobs.
   - 5-minute timeout — the script is ~3s in practice; 5 minutes is generous slack for cold-start + network jitter.
   - Defaults in `scripts/post-deploy-smoke.sh` already point at the prod URL + cloud project, so no env wiring needed.
2. **Test for the workflow file (`scripts/test-ci-config.test.ts`):** parses the YAML with `js-yaml`, asserts the job's shape. Lives outside `apps/web` because it audits root-level CI config.
3. **Run it locally first:** `pnpm smoke` once before pushing. The CI invocation should pass on the first push to `main`.

## Out of scope
- Slack/email notification on smoke fail (Phase-9 polish; for v1 the CI red badge is enough).
- Pre-merge smoke against the *preview* deployment — Vercel preview URLs are per-PR and ephemeral; smoking them adds value but adds matrix complexity. Defer.
- Smoke retry-on-flake. The script is deterministic against stable infra; if a single run flakes, re-run the workflow.
