# Request 8.3 — Frontend Error Capture to Supabase

## Objective
Persist uncaught client errors so I can triage what Ben hits when he opens an issue or messages me. Today the only signal a frontend bug ever generates is "the page broke" — there's no log, no stack, no breadcrumb.

Scope: a `client_error_log` table + RLS, an `ErrorBoundary` around the route tree that writes to it, a `window.onerror` + `unhandledrejection` listener that does the same, and a tiny query the user (logged in as admin) can run to see recent errors.

Deliberately *not* a third-party SaaS (Sentry, LogRocket) for v1: one user, one repo, costs and configuration are higher than the value at this scale. When Repeaty has 100+ users, swap in Sentry.

## GREEN Phase

1. **Migration `supabase/migrations/0019_client_error_log.sql`:**
   ```sql
   create table public.client_error_log (
     id uuid primary key default gen_random_uuid(),
     user_id uuid not null references auth.users(id) on delete cascade default auth.uid(),
     message text not null,
     stack text null,
     route text null,
     app_version text null,
     user_agent text null,
     viewport_w int null,
     viewport_h int null,
     extra jsonb null,
     created_at timestamptz not null default now()
   );

   alter table public.client_error_log enable row level security;

   create policy "users can insert their own errors"
     on public.client_error_log for insert
     with check (auth.uid() = user_id);

   -- No general SELECT policy — admins read via service-role from Studio,
   -- or via a SECURITY DEFINER RPC if/when an in-app /admin/errors view ships.

   create index idx_client_error_log_user_created
     on public.client_error_log (user_id, created_at desc);
   ```
2. **`apps/web/src/lib/error-log.ts`:**
   - `logClientError(payload: ClientErrorPayload): void` — fire-and-forget, wraps `supabase.from('client_error_log').insert(...)` in try/catch, never propagates.
   - `scrubPayload(p)` — strips fields matching `/(password|token|jwt|api[_-]?key)/i` from any object value, replaces matches against `/sk-(ant-)?[a-zA-Z0-9]+/` with `<scrubbed>` in strings.
   - Hard-cap `stack` at 8 KB and `extra` at 4 KB to avoid filling the table with one error.
3. **`apps/web/src/components/ErrorBoundary.tsx`:** standard React error boundary; calls `logClientError` in `componentDidCatch`; renders a small fallback UI with a "Reload" button. Wrap `<App>` in `apps/web/src/main.tsx`.
4. **`apps/web/src/lib/useGlobalErrorListeners.ts`:** registers `window.addEventListener('error', ...)` + `'unhandledrejection'`; logs each event. Cleanup in the effect's return.
5. **Build-time `__APP_VERSION__`:** Vite `define` config exposes `import.meta.env.VITE_APP_VERSION` from `git rev-parse --short HEAD`. Falls back to `"dev"` in non-CI builds.
6. **Drift updates:**
   - `references/schema.md`: new § `client_error_log` after `tier_change_log`.
   - `audits/debt.md` § Resolved: not applicable (this isn't a deferred item; it's net-new).

## Out of scope
- In-app `/admin/errors` triage page — Studio is enough for v1. (Pull this forward as a follow-up if Ben actually breaks something.)
- Source-map upload to map minified stacks to source. Stacks land minified; live commit hash + line numbers are enough to grep the source tree.
- Sentry integration — when Repeaty has 100+ users, do this. Today: not worth the dependency.
- Edge-Function-side error capture. The four functions already log structured JSON to stdout, and Supabase Dashboard's Logs view is queryable. Adding a `server_error_log` table would be useful but doubles scope; defer to a Phase-9 polish if needed.

## Risks
- **Self-amplifying error loop.** A bug in `logClientError` itself triggers a render error → ErrorBoundary calls `logClientError` → boom. Mitigate with the helper's own try/catch and a single in-memory rate limiter: drop on the floor after 5 inserts in 60s.
- **PII in stack frames.** Function arguments captured in stack traces can include user input. Out of scope to fully strip; the secret-pattern scrubber is the v1 line. Document this in `references/security-landmines.md`.
